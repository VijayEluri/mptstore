5   /**
6    * Provides RDBMS-specific DDL string(s) for table management functions.
7    *
8    * <p>
9    *   The map table should have columns pKey (which holds an auto-incremented
10   *   integer) and p (which holds the actual predicate in N-Triples format).
11   * </p>
12   * <p>
13   *   The SO (predicate) tables should each have columns s and o,
14   *   for holding the subject and object strings in N-Triples format.
15   * </p>
16   * <h2>Note on Datatypes</h2>
17   * <p>
18   *   Typically the s, p, and o datatypes be declared as TEXT or CLOB in the
19   *   underlying database, but they can also be defined as varchars if the
20   *   data set is known to fit within some pre-determined range.  The
21   *   type should be such that comparisons are case sensitive,
22   *   as N-Triples is a case sensitive format.
23   * </p>
24   * <h2>Note on Character Encoding</h2>
25   * <p>
26   *   Note: The database need only be capable of storing 7-bit ASCII
27   *   because the RDF is stored in N-Triples format.  In fact, it will
28   *   likely perform better if the database is defined as such because
29   *   more data will fit in less space if characters only take up 1 byte
30   *   in the database.
31   * </p>
32   *
33   * @author cwilper@cs.cornell.edu
34   */

44      List<String> getCreateMapTableDDL(String table);
